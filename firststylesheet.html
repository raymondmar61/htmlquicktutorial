font-size values are the length number and length unit, percentage number, xx-small, x-small, small, medium, large, x-large, xx-large, smaller or larger is the relative to the surrounding text. Default is medium. The preferred values for font-size in contemporary web design are the relative length units em and rem and percentage values.

To maintain the browser's default size, set the font-size of the root element to 100%. html {font-size: 100%;} That sets the basis for relative sizing. Because the default font size for all modern browsers is 16 pixels, we'll assume our base size is 16 pixels going forward.

The rem unit stands for root em relative to the size of the root or html element. Root size is 16 pixels. rem equals 16 pixels. 1.5rem is 1.5 * 16pixels = 24pixels. h1 {font-size=1.5rem;}

Em units are based on the font size of the current element. Em is relative to the inherited size for the element. h1 inherited the default 16 pixel font size from the root. 2em is 2 * 16pixels = 32pixels. p {font-size: 2em;}. There are a few snags with em. 1 rounding errors and inconsistency how browsers render in em. 2 em based on the inherited size of the element, which means that their size is based on the context in which they are applied. 3 If you have elements nested several layers deep, the size increase or decrease compounds, which can create problems. With many layers of nesting, text may end up being way too small. Use rem.

The id attribute gives an element a unique identifying name or id reference. The id attribute can be used with any element. Use id selectors to target elements by their id values. Use the hashtag # to identify id selectors. id values must be unique in the document. It's acceptable to omit the html element name. #paragraphid {font-family: "Times New Roman"; color: black !important;} and p#paragraphid {font-family: "Times New Roman"; color: black !important;} are the same. If you want an id attribute for a specific html element only, then mentioned the html element after the id name; for example, #idappliestoaelementonly a {text-decoration: none;}.

There are other contextual selectors. RM: it's briefly mentioned in a container box. The other three are child selectors, next-sibling selectors, and subsequent-sibling selectors.
A child selector targets only the direct children of a given element indicated with the greater-than symbol (>). The following rule affects emphasized text only when it is directly contained in a p element. An em element inside a link (a) within the paragraph would not be affected. p > em {font-weight: bold;}.
A next-sibling selector targets an element that comes directly after another element with the same parent. It is indicated with a plus (+) sign. This rule gives special treatment to paragraphs that follow an h1. h1 + p {font-style: italic;}
A subsequent-sibling selector selects an element that shares a parent with the specified element and occurs after it in the source order. They do not need to follow one another directly. The following rule selects any h2 that both shares a parent element (such as a section or article) with an h1 and appears after it in the document. h1 ~ h2 {font-weight: normal;}
