Every element generates a box for which properties such as width, height, padding, borders and margins are applied. The margin is the distance between the outer edge and the content. The padding is the distance between outer content edge and the content area. Padding is like the air inside an unopened potato chips bag.

width: length, percentage, or auto. Auto is default. The width and height applies to block-level elements and non-text inline elements such as images. Inline text invalid such as an anchor a tag or bold strong tag.

height: length, percentage, or auto. Auto is default. The width and height applies to block-level elements and non-text inline elements such as images. Inline text invalid such as an anchor a tag or bold strong tag.

box-sizing: content-box or border-box. Content-box is default.

Default content-box. Box width 500 pixes, height 150 pixels, 20 pixels padding, 5 pixels border, 20 pixels margin. Width and height applied to the content area only. The entire box or element box is 550 pixels. The total width is 570 pixels=20px margin on left+5px margin on left+10px padding on left+500px width+10px padding on left+5px margin on right+20px margin on right.

Border-box model. Box width 500 pixes, height 150 pixels, 20 pixels padding, 5 pixels border, 20 pixels margin. Width and height applied to the content area only. Total visible box width 500px. The total width is 540 pixels=20px margin on left+500px width+20px margin on right.
Border-box good for specifying widths in percentages. For example, make two columns 50% wide.
Many developers set everything to border-box model declared in the html element style sheet html{box-sizing: border box;}

overflow: visible, hidden, scroll, or auto. Visible is default. An element is sized too small for its content, specify what happens when the content doesn't fit.

padding, padding-top, padding-right, padding-bottom, patting-left: length or percentage. Default is 0. Top padding 2em, right padding 4em, bottom padding 2em, left padding 4em. Padding can be specify by em, px, percentage of the width of the parent element. Em is relative to the inherited size for the element. The rem unit stands for root em relative to the size of the root or html element. Root size is 16 pixels. rem equals 16 pixels. 1.5rem is 1.5 * 16pixels = 24pixels. Em units are based on the font size of the current element. Em is relative to the inherited size for the element. h1 inherited the default 16 pixel font size from the root. 2em is 2 * 16pixels = 32pixels.

Use padding and specify the individual properties top, right, bottom and left 2em 4em 2em 4em.

Use padding and specify the individual properties top and bottom and right and left 2em 4em.

Use padding and specify the individual properties all sides 15px.

border-style, border-top-style, border-right-style, border-bottom-style, border-left-style: none, solid, hidden, dotted, dashed, double, groove, ridge, inset, outset. None is default.

border-width, border-top-width, border-right-width, border-bottom-width, border-left-width: length, thin, medium, thick. Default is medium. Need a border-style to display a border obviously.

border-color, border-top-color, border-right-color, border-bottom-color, border-left-color: color name or RGB or hexadecimal, transparent. Default is the color property for the element.

Combine border style, width, and color

Rounded corners use border-radius. border-radius, border-top-left-radius, border-top-right-radius, border-bottom-right-radius, border-bottom-left-radius.

Elliptical corners use border-radius, border-top-left-radius, border-top-right-radius, border-bottom-right-radius, border-bottom-left-radius. The two numbers specify the horizontal radius and the second for the vertical radius. The shorthand is border-radius: horizontal radius all corners and vertical radius all corners. For example border-radius: 60px/40px; Can shorthand from top left to bottom left. For example, border-radius: 36px 40px 60px 20px/12px 10px 30px 36px;

Use border-image for a picture border. The shorthand border-image property that combines the individual properties in the following syntax: border-image: source slice / width / outset repeat; Slice divides the image into nine sections using offset measurements in px. Width specifes the border area in px. Outset pushes the border away from the content by the specified amount. Repeat specifies whether the image should stretch or repeat along the sides. For example, border-image: url(fancyframe.png) 55 fill / 55px / 25px stretch;

Outlines are like borders. Outlines are not calculated in the width of the element box. Outlines are drawn on the outer edge of the border if specified and overlaps the margin. Good for checking design. outline outline-style outline width outline color; outline-style auto, solid, none, dotted, dashed, double, groove, ridge, inset or outset default is none; outline-width length, thin, medium and thick default is medium; outline color color name or RGB or hexadecimal or invert default is invert which is the inverse of the background color, outline-offset length default is 0 which is the outline moves beyond the border by a specified length.

A margin is space added on the outside of the border. Margins keep elements from bumping into each other. The side-specific and shorthand margin is the same for padding. style="margin: 1px 2px 3px 4px; If two values are provided, the first value is top and bottom and the second value is right and left. Numbers are in em, px, and percentage.

margin-top, margin-right, margin-botton, margin-left: length, percentage, or auto. Auto is default. Numbers are in em, px, and percentage. The top and bottom margins of elements next to each other collapse. The adjacent bottom margin and its top margin don't accumulate. The largest value of the top is used. If the top element has a bottom margin of 4em and bottom element has a top margin of 2em, the resulting margin space between elements does not add up to 6ems. The margins collapse and the resulting margin between the paragraphs will be 4em, the largest specified value. The top and bottom margins don't collapse is floated or absolute position elements discussed in chapter 15. Margins on the left or right sides never collapse.

margin-top: 2em; margin-right: 250px; margin-bottom: 1em; margin-left: 4em; border: 2px solid red; background: #e2f3f5;

margin: 0 20%; border: 3px solid red; background: #e2f3f5; 0% margin top and 0% margin bottom. 20% margin right and 20% margin left. Numbers are not divided by each side top and bottom, and right and left

Top margin and bottom margin can be used for inline text elements. It doesn't add vertical space above and space below. The height of the line is unchanged. Left margin and right margin to inline text elements are valid.

Can specify negative values for margins. Margins move in the opposite direction.

The box-shadow adds a drop shadow around the element box excluding the margin. box-shadow: horizontal offset vertical offset blur distance spread distance color inset, none. Default is none.
