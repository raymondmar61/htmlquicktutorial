The browser uses for page elements renders the elements in the order in which they appear in the HTML file, stacking block-level elements and allowing inline elements to fill their parent blocks left to right. Another technique you can use (besides floating elements) to break out of the default flow is positioning one or more elements yourself, using the CSS position property combined with other CSS properties. element {position: static|relative|absolute|fixed; top: measurement|percentage|auto; right: measurement|percentage|auto; bottom: measurement|percentage|auto; left: measurement|percentage|auto; z-index: integer|auto;}.

A quick summary of position: static-Ignores the offset properties (this is the default positioning used by the browser). relative-Positions the element offset from its default position while keeping the element’s default position within the page flow. absolute-Positions the element at a specific place within the nearest ancestor that has a nonstatic position while removing the element from the page flow. fixed-Positions the element at a specific place within the browser viewport while removing the element from the page flow. If the position is relative, the element's default position remains in the normal page flow. The element is shifted. If value is top, then the element is shifted down. If value is right, then the element is shifted from the right. If value is bottom, then the element is shifted up. If value is left, the element is shifted from the left.

Relative positioning shifts an element out of its default position while preserving the element's original space in the page flow. This can cause page weirdness. For example, if you set the top property, the element shifts down. This leaves a gap where the element would have been, which can look odd.

An absolute position takes the element out of the default page flow and positions the element with respect to its nearest nonstatic ancestor or positioned ancestor. The ancestor moves up the hierarchy to the lement's parent, grandparent, and so on. The ancestor is the first element its position property set other than static. If no ancestor is found, the element's absolute position is set with respect to the browser's content area. If the position is top, the element is moved down from the ancestor's top edge. If the position is right, the element is moved left from the ancestor's right edge. If the position is bottom, the element is moved up from the ancestor's bottom edge. If the position is left, the element is moved right from the ancestor's left edge.

Absolute positioning moves an element from its default position, but doesn't preserve the its original space in the page flow. The element's new position is set with respect to the nearest ancestor in the hierarchy that has a non-static position, or the browser window if no such ancestor exists. Intro
